# Програмне забезпечення енергетичного менеджменту (ЛР №1)

Даний проєкт присвячений розробці системи збору та збереження даних для об'єкта енергоменеджменту в рамках навчальної дисципліни.

## Опис об'єкта (Варіант №4)
Об'єктом дослідження є **Готель**, що функціонує в режимі 24/7.
* **Площа**: 2500 м².
* **Потужність СЕС**: 120 кВт.
* **Ємність батареї**: 100 кВт*год.
* **Локація**: м. Київ (враховано кліматичні дані 2025 року).

## Структура сховища
База даних реалізована в **PostgreSQL** за Треком А (реляційна архітектура). 
Система підтримує трирівневий облік енергоспоживання:
1. **1 рівень**: Загальний ввід об'єкта.
2. **2 рівень**: Вузлові точки (Харчоблок, Житловий корпус, Технічні системи).
3. **3 рівень**: Кінцеві споживачі (Освітлення, обладнання кухні, номери).

## Структура файлів проекту
* `setup.sql`: Скрипт створення таблиць, індексів, обмежень цілісності та View для рівнів обліку.
* `generate_data.py`: Python-скрипт для генерації 8760 годин реалістичних даних (погода, споживання, генерація СЕС).
* `queries.sql`: Набір обов'язкових аналітичних запитів (споживання за періодами, аналіз тарифних зон, виявлення аномалій).

## Встановлення та запуск
1. Розгорніть структуру бази даних за допомогою `setup.sql`.
2. Встановіть необхідні бібліотеки Python: `pip install pandas numpy`.
3. Запустіть `generate_data.py` для створення CSV-файлів з даними.
4. Імпортуйте дані в PostgreSQL за допомогою команди `COPY` (шляхи до файлів вказуються в `setup.sql`).
5. Використовуйте `queries.sql` для перевірки працездатності системи та отримання аналітики.
