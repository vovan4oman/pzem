# Програмне забезпечення енергетичного менеджменту (ЛР №1)

 Даний проєкт присвячений розробці системи збору та збереження даних для об'єкта енергоменеджменту в рамках навчальної дисципліни[cite: 1].

## Опис об'єкта (Варіант №4)
 Об'єктом дослідження є **Готель**, що функціонує в режимі 24/7[cite: 483].
*  **Площа**: 2500 м²[cite: 483].
*  **Потужність СЕС**: 120 кВт[cite: 483].
*  **Ємність батареї**: 100 кВт*год[cite: 483].
* **Локація**: м.  Київ (враховано кліматичні дані 2025 року)[cite: 452, 461].

## Структура сховища
 База даних реалізована в **PostgreSQL** за Треком А (реляційна архітектура)[cite: 46]. 
 Система підтримує трирівневий облік енергоспоживання[cite: 44]:
1.  **1 рівень**: Загальний ввід об'єкта[cite: 68].
2.  **2 рівень**: Вузлові точки (Харчоблок, Житловий корпус, Технічні системи)[cite: 68].
3.  **3 рівень**: Кінцеві споживачі (Освітлення, обладнання кухні, номери)[cite: 68].

## Структура файлів проекту
*  `setup.sql`: Скрипт створення таблиць, індексів, обмежень цілісності та View для рівнів обліку[cite: 105, 106].
*  `generate_data.py`: Python-скрипт для генерації 8760 годин реалістичних даних (погода, споживання, генерація СЕС)[cite: 77, 80].
*  `queries.sql`: Набір обов'язкових аналітичних запитів (споживання за періодами, аналіз тарифних зон, виявлення аномалій)[cite: 91, 103].

## Встановлення та запуск
1. Розгорніть структуру бази даних за допомогою `setup.sql`.
2. Встановіть необхідні бібліотеки Python: `pip install pandas numpy`.
3. Запустіть `generate_data.py` для створення CSV-файлів з даними.
4. Імпортуйте дані в PostgreSQL за допомогою команди `COPY` (шляхи до файлів вказуються в `setup.sql`).
5. Використовуйте `queries.sql` для перевірки працездатності системи та отримання аналітики.
